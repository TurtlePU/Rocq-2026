MapsТотальные и частичные словари

_Мапы_ (или _словари_) постоянно встречаются как в обычном программировании, так и в теории языков программирования; в последующих главах они нам очень пригодятся.
Одновременно с этим, на них будет удобно отработать техники, представленные в предыдущих главах, включая построение структур данных с помощью функций высшего порядка (как в Basics и Poly) и использование рефлексии для упрощения доказательств (главы Logic, IndProp).
Мы определим две разновидности словарей: _тотальные_ включают элемент "по умолчанию", возвращаемый, если ключа нет в словаре; _частичные_ возвращают option, чтобы иметь возможность сообщать об ошибке. Частичные словари определяются как тотальные, где в качестве элемента по умолчанию используется None.

From Stdlib Require Import Arith.

Идентификаторы

Для того, чтобы определить тип словарей, нам нужно определиться с типом ключей, по которым осуществляется доступ к значениям в словаре. В Lists.v мы определяли тип id; здесь и далее мы будем использовать тип string из стандартной библиотеки.
Для сравнения строчек мы используем функцию eqb из стандартного модуля String.
Check String.eqb_refl :
  ∀ x : string, (x =? x)%string = true.

Мы будем пользоваться некоторыми базовыми свойствами равенства строк...
Check String.eqb_eq :
  ∀ n m : string, (n =? m)%string = true ↔ n = m.
Check String.eqb_neq :
  ∀ n m : string, (n =? m)%string = false ↔ n ≠ m.
Check String.eqb_spec :
  ∀ x y : string, reflect (x = y) (String.eqb x y).

Тотальные словари

Нашей главной задачей в этой главе будет построить определение частичных отображений, которое будет похоже по поведению на словари из главы Lists, плюс доказать леммы об их поведении.
Правда, в этот раз для построения словарей мы будем использовать _функции_ вместо списка пар ключ-значение. Преимущество такого представления в том, что оно предлагает более "экстенсиональный" взгляд на словари: два словаря, которые отвечают на запросы одинаково, будут представлены одной и той же функцией, а не просто "эквивалентными" списками. Это упростит доказательства, использующие словари.

Мы строим частичные словари в два этапа. На первом мы определяем тип _тотальных словарей_, которые возвращают значение по умолчанию, когда мы запрашиваем ключ, отсутствующий в словаре.
Definition total_map (A : Type) := string → A.
По сути, тотальный словарь элементов типа A -- это просто функция, которая по stringам возвращает элементы типа A.

Функция t_empty по элементу-по-умолчанию возвращает пустой тотальный словарь; этот словарь для любого ключа возвращает этот самый элемент по умолчанию.
Definition t_empty {A : Type} (v : A) : total_map A :=
  (fun _ ⇒ v).

Функция обновления словаря будет поинтереснее. Как и ожидается, она получает словарь m, ключ x и значение v и возвращает новый словарь, который ассоциирует x с v, а остальные ключи ассоциированы так же, как в m. Новизна здесь в том, что мы достигаем этого эффекта, как бы оборачивая новую функцию вокруг старой.
Definition t_update {A : Type} (m : total_map A)
                    (x : string) (v : A) :=
  fun x' ⇒ if String.eqb x x' then v else m x'.
Это определение -- хороший пример программирования функций высшего порядка: t_update получает _функцию_ m и возвращает новую функцию fun x' ⇒ ..., которая ведёт себя как нам хочется.

Например, мы можем построить словарь, связывающий stringи с boolями, где "foo" и "bar" отображаются в true, а все остальные ключи -- в false, вот так:
Definition examplemap :=
  t_update (t_update (t_empty false) "foo" true)
           "bar" true.

Далее, давайте введём несколько нотаций для работы со словарями.
Во-первых, мы используем следующую нотацию для представления пустого тотального словаря со значением по умолчанию.
Notation "'__' '!->' v" := (t_empty v)
  (at level 100, right associativity).
Example example_empty := (__ !-> false).

Далее мы вводим нотацию для расширения существующего словаря новым присваиванием.
Notation "x '!->' v ';' m" := (t_update m x v)
                                (at level 100, v constr at level 100, right associativity).

Теперь examplemap можно определить вот так:
Definition examplemap' :=
  ( "bar" !-> true;
    "foo" !-> true;
    __ !-> false
  ).

Когда мы будем использовать словари в последующих главах, нам потребуется несколько фундаментальных фактов об их поведении.
Lemma t_apply_empty : ∀ (A : Type) (x : string) (v : A),
  (__ !-> v) x = v.
Proof.
  (* ЗАПОЛНИТЕ ЗДЕСЬ *) Admitted.

Lemma t_update_eq : ∀ (A : Type) (m : total_map A) x v,
  (x !-> v ; m) x = v.
Proof.
  (* ЗАПОЛНИТЕ ЗДЕСЬ *) Admitted.

Theorem t_update_neq : ∀ (A : Type) (m : total_map A) x1 x2 v,
  x1 ≠ x2 →
  (x1 !-> v ; m) x2 = m x2.
Proof.
  (* ЗАПОЛНИТЕ ЗДЕСЬ *) Admitted.

Lemma t_update_shadow : ∀ (A : Type) (m : total_map A) x v1 v2,
  (x !-> v2 ; x !-> v1 ; m) = (x !-> v2 ; m).
Proof.
  (* ЗАПОЛНИТЕ ЗДЕСЬ *) Admitted.

Theorem t_update_same : ∀ (A : Type) (m : total_map A) x,
  (x !-> m x ; m) = m.
Proof.
  (* ЗАПОЛНИТЕ ЗДЕСЬ *) Admitted.

Theorem t_update_permute : ∀ (A : Type) (m : total_map A)
                                  v1 v2 x1 x2,
  x2 ≠ x1 →
  (x1 !-> v1 ; x2 !-> v2 ; m)
  =
  (x2 !-> v2 ; x1 !-> v1 ; m).
Proof.
  (* ЗАПОЛНИТЕ ЗДЕСЬ *) Admitted.

Частичные словари

Наконец, мы определяем _частичные словари_ поверх тотальных. Частичный словарь элементов типа A -- просто тотальный словарь элементов типа option A, где элемент по умолчанию равен None.
Definition partial_map (A : Type) := total_map (option A).
Definition empty {A : Type} : partial_map A :=
  t_empty None.
Definition update {A : Type} (m : partial_map A)
           (x : string) (v : A) :=
  (x !-> Some v ; m).

Мы вводим похожую нотацию для частичных словарей:
Notation "x '⊢>' v ';' m" := (update m x v)
  (at level 0, x constr, v at level 200, right associativity).
Мы также можем опустить последний случай: соответствующий словарь пустой.
Notation "x '⊢>' v" := (update empty x v)
  (at level 0, x constr, v at level 200).
Definition examplepmap :=
  ("Church" ⊢> true ; "Turing" ⊢> false).

Теперь мы просто переносим все базовые леммы о тотальных словарях.
Lemma apply_empty : ∀ (A : Type) (x : string),
  @empty A x = None.
Lemma update_eq : ∀ (A : Type) (m : partial_map A) x v,
  (x ⊢> v ; m) x = Some v.

Theorem update_neq : ∀ (A : Type) (m : partial_map A) x1 x2 v,
  x2 ≠ x1 →
  (x2 ⊢> v ; m) x1 = m x1.
Lemma update_shadow : ∀ (A : Type) (m : partial_map A) x v1 v2,
  (x ⊢> v2 ; x ⊢> v1 ; m) = (x ⊢> v2 ; m).
Theorem update_same : ∀ (A : Type) (m : partial_map A) x v,
  m x = Some v →
  (x ⊢> v ; m) = m.
Theorem update_permute : ∀ (A : Type) (m : partial_map A)
                                x1 x2 v1 v2,
  x2 ≠ x1 →
  (x1 ⊢> v1 ; x2 ⊢> v2 ; m) = (x2 ⊢> v2 ; x1 ⊢> v1 ; m).

И последнее: полезно ввести понятие включения частичных словарей, утверждающее, что все записи одного словаря также присутствуют в другом:
Definition includedin {A : Type} (m m' : partial_map A) :=
  ∀ x v, m x = Some v → m' x = Some v.
Тогда мы можем показать, что расширение словарей сохраняет отношение включения, то есть:
Lemma includedin_update : ∀ (A : Type) (m m' : partial_map A)
                                 (x : string) (vx : A),
  includedin m m' →
  includedin (x ⊢> vx ; m) (x ⊢> vx ; m').
Это свойство достаточно полезно для рассуждения о языках с переменными -- например, просто типизированного лямбда-исчисления, которое мы увидим на одном из следующих занятий, где словари используются для отслеживания того, какие переменные определены в текущей области видимости.
